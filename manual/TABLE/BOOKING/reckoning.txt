RECKONING - таблица заявок

id - номер заявки
date - дата добавления заявки
sum - сумма заявки
1C - номер счета в 1С (идентичен id) -- старый
id_user - id менеджера (чья заявка)
turist - id туриста
active - по умолчанию 0 (не в архие); 1 - в архиве, 2 - заявка обработанна (её нельзя изменить)
status - статус заявки (неоплачена, оплачена, аннулирована и тд)
rest - отдыхающие по заявке (id туристов , через запятую)
prepay - предоплата (сумма)
id_reg - id региона
id_obj - id объекта
date_z - дата заезда
status_san - статус оплаты в санаторий (0 - не оплачено в санаторий, 2 - разрешение на оплату в санаторий, 1 - оплачено в санаторий, 3 - предоплата в санаторий)
number_turist - количество туристов заявки
date_opl_san - дата оплаты счета санатория
pay_number - номер счета оплаты в санаторий
prepay_number - номер счета предоплаты в санаторий
payer - плательщик
agency - id агенства
id_com - id комиссии (для агенств)
note - примечание к заявке
id_dis - id скидки (для клиентов)
id_services - id услуг санатория (питание, лечение ...)
changes - JSON массив с изменениями (для ЛК туриста, отмечается красным что именно изменено)
reward - вознаграждение по заявке, установленное вручную
note_bid - примечание к заявке для объектов
correction - корректировка
source - источник (с рекламы, с соц сетей, с рассылки)
promo_code - промокод
form_booking - способ бронирования (квота мест, заказ звонка)
