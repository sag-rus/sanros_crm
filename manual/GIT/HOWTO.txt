папка CRM - рабочая версия, CRM_dev - изменения
Две ветки: master и dev, на рабочей CRM всегда должна быть активна ветка master, на ветке dev - последние изменения

На какой ветке находимся: git branch
Меняем ветку: git checkout master (или dev)

Добавить новые файлы в GIT для отслеживания: git add --all
Сохранить изменения: git commit -am "Комментарий, что изменили"
Отправить изменения из CRM_dev (ветка dev) на CRM (ветка dev): git push origin dev
Применить изменения в CRM (активная ветка должна быть master): git merge dev
